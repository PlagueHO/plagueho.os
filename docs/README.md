# Documentation

This directory contains reference documentation, guides, and other materials used across repositories.

## Structure

```
docs/
├── README.md         # This file
├── conventions/      # Coding conventions and standards
├── guides/           # How-to guides and tutorials
└── reference/        # Reference material and cheat sheets
```

## Categories

### Conventions

Standards and conventions for code style, naming, commit messages, branching strategies, and other development practices.

### Guides

Step-by-step guides for common tasks and workflows.

### Reference

Quick reference material, cheat sheets, and external resource links.

## Adding Documentation

1. Place documentation in the appropriate subdirectory
2. Use Markdown format (`.md`) for all documents
3. Include a title and brief description at the top of each document
4. Link to related documents where applicable
