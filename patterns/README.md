# Patterns

This directory contains reusable development patterns, templates, and architectural blueprints used across repositories.

## Structure

```text
patterns/
├── README.md             # This file
├── github-actions/       # Reusable GitHub Actions workflow patterns
├── azure/                # Azure infrastructure and deployment patterns
├── dotnet/               # .NET application patterns
├── python/               # Python application patterns
└── general/              # General-purpose patterns applicable to any tech stack
```

## What is a Pattern?

A pattern is a reusable, opinionated solution to a commonly encountered problem. Patterns in this repository are:

- **Documented**: Each pattern includes a README explaining the problem it solves, when to use it, and how to adapt it
- **Tested**: Patterns have been validated in real projects
- **Parameterized**: Where possible, patterns use variables/tokens to make them easy to customize

## Pattern Structure

Each pattern should include:

1. **README.md** — Description, use cases, and instructions
2. **Template files** — The actual pattern files (code, config, etc.)
3. **Example** — A concrete example of the pattern in use (optional)

## Adding a Pattern

1. Create a new subdirectory under the appropriate technology folder
2. Add a `README.md` describing the pattern
3. Add the template files
